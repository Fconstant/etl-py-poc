# Fresh to Data Engineering

User is a traditional fullstack developer (Node/React/Postgres background), completely new to data engineering and GCP. This sets the floor: all BigQuery teaching must bridge from their Postgres mental model. Do not assume familiarity with OLAP, columnar storage, data warehousing concepts, or GCP IAM/credentials.

Implications: First lessons should explicitly compare Postgres concepts to BigQuery equivalents. Avoid jumping into BigQuery SQL features without establishing the OLTP vs OLAP distinction first.
